# H-NS & StpA Extraction Pipeline

A modular python-based pipeline to extract and classify H-NS and StpA proteins from bacterial genome assemblies, predicting whether they reside on the chromosome or plasmids.

## Features
* **ORF Finding:** Rapid gene prediction using `pyrodigal`.
* **HMM Scanning:** Profile HMM search with `pyhmmer` against the H-NS Pfam profile (`PF00816.hmm`).
* **Paralog Classification:** Zero-I/O local alignment using Biopython's `PairwiseAligner` to automatically distinguish between H-NS and StpA.
* **Genomic Location:** Plasmid vs. Chromosome origin prediction for contigs using `Platon` via Apptainer.
* **Clustering:** CD-HIT clustering at 100% identity via Apptainer to isolate unique sequence variants.
* **Slurm Orchestration:** Full HPC resource management script utilizing Astral's `uv` for package management.

---

## Dependencies & Requirements

* **Python 3.8+** (System python base module)
* **Apptainer** (Required for containerized CD-HIT and Platon execution)
* **Platon Database:** Located at `/project/GROUP-MOLEPI/databases/platon_db/db` (HPC specific)
* **Apptainer SIF Images:** Placed in `/project/GROUP-MOLEPI/apptainer/` (`platon.sif` and `cd-hit.sif`)

## Installation & Setup

All required Python libraries (`biopython`, `pyrodigal`, `pyhmmer`, `pandas`, `uv`) are installed dynamically inside a local virtual environment. You do not need to install them manually.

To setup the environment and download the H-NS HMM profile:
```bash
# Clone/navigate to the pipeline directory
cd /project/GROUP-MOLEPI/HNS/

# Download the HMM profile from the Pfam/InterPro API
python3 src/hns_extractor.py --download-hmm
```

---

## Usage Guide

### 1. Locally/Interactively

To run the pipeline interactively (e.g., on a login node or interactive session):
```bash
# Activate the virtual environment
source venv/bin/activate

# Execute the pipeline
python3 src/hns_extractor.py \
  --input-list test_input/test_samples.tsv \
  --output-tsv test_report.tsv \
  --output-dir test_extracted_hns \
  --output-unique \
  --cpus 4
```

### 2. On a Slurm Cluster (HPC)

Submit the orchestrated batch script `run_hns_pipeline.sh`:
```bash
sbatch run_hns_pipeline.sh <input_tsv> <output_tsv> <output_dir>
```

**Example:**
```bash
sbatch run_hns_pipeline.sh \
  test_input/test_samples.tsv \
  test_report.tsv \
  test_extracted_hns
```

## Input File Formats

### Genome assembly list (`test_samples.tsv`):
A Tab-Separated Values file containing `Sample_Name` in the first column, and the absolute path to the assembly FASTA file in the second column.
```tsv
SAMEA114094068_30EN_local	/project/GROUP-MOLEPI/HNS/test_input/SAMEA114094068_30EN_local.fasta
```

## Output Files

Upon successful completion, the pipeline outputs:
* **`test_report.tsv`**: Full Bakta-style report with genomic coordinates, H-NS vs. StpA classification, identity/coverage metrics, and plasmid/chromosome location.
* **`test_extracted_hns/`**:
  * `{Sample_Name}.faa`: Raw amino acid FASTA file containing all extracted proteins for the sample.
  * `{Sample_Name}_unique.faa`: Non-redundant FASTA file (100% identity representatives) for the sample generated by CD-HIT.
  * `{Sample_Name}_unique.faa.clstr`: CD-HIT clustering file for the sample.
  * `{Sample_Name}.contigs.fasta`: FASTA nucleotide file containing contigs carrying H-NS/StpA hits.
  * `{Sample_Name}_platon/`: Comprehensive Platon prediction raw reports.
  * `{Sample_Name}.done`: Empty sentinel file indicating that H-NS extraction and Platon prediction for this sample completed successfully.
  * `{Sample_Name}_hits.json`: Cached scanner results to allow instant resume execution.

---

## Checkpointing & Resume Mode

To facilitate running on large batches (e.g., 1,400+ genomes) on high-performance compute clusters, the pipeline has a built-in checkpoint/resume mechanism:
* **H-NS Scanner Cache:** When starting, the pipeline checks if `{output_dir}/{Sample_Name}.done` exists. If present, it loads the sample's hits directly from the `{Sample_Name}_hits.json` cache in milliseconds, bypassing gene prediction and HMMER scans.
* **Platon Cache:** Platon is skipped for any sample that already has a non-empty `{Sample_Name}_platon/{Sample_Name}.contigs.tsv` output file.
* **To Clean Run:** Simply target a new output directory or empty the existing one.

---

## Memory Concurrency Guard

* **Auto-scaling Workers:** Platon executions run concurrently to optimize performance. To prevent Out-Of-Memory (OOM) failures under Slurm, the pipeline reads `SLURM_MEM_PER_NODE` and automatically caps the concurrent Platon worker count:
  $$\text{Max Platon Workers} = \max\left(1, \frac{\text{SLURM\_MEM\_PER\_NODE}}{2 \text{ GB}}\right)$$
  If the memory allocated is too low for the requested CPUs, the pipeline downscales parallel worker count dynamically to run safely within limits.
